# Travel Map Redesign — Design Spec
Date: 2026-06-28

## Overview

TravelContent 컴포넌트를 리팩토링하여 검색 기반 핀 추가, 사이드바 여행 목록, fit bounds 지도 이동, 핀 상세 패널(사진 포함)로 개선한다. 데이터 구조와 API는 변경하지 않는다.

---

## Layout

### Desktop
- **좌측 사이드바 (240px)**: 여행 목록 아코디언 + 장소 목록 + "+ 새 여행" 버튼
- **우측 지도**: 나머지 전체 공간, 상단에 검색창 오버레이

### Mobile
- **상단**: 여행 드롭다운 + "+ 새 여행" 버튼
- **지도**: 전체 화면, 상단에 검색창 오버레이
- **바텀시트**: 선택된 여행의 장소 목록 (드래그 업)

---

## 사이드바 (Desktop)

- 여행 목록을 아코디언 형태로 표시
- 여행 클릭 → 펼쳐서 해당 여행 장소 목록 표시
- 장소 클릭 → 해당 핀으로 지도 이동 + 상세 패널 열림
- 각 여행 옆 ✏️(수정), 🗑(삭제) 버튼
- "+ 새 여행" 버튼으로 TripFormSheet 열기

---

## 바텀시트 (Mobile)

- 선택된 여행의 장소 목록 표시
- 장소 탭 → 해당 핀으로 지도 이동 + 상세 패널 열림
- 드래그로 높이 조절 (접힘/펼침)

---

## 핀 동작

- 여행 선택 시 해당 여행의 모든 장소가 보이도록 지도 범위 자동 조정 (kakao LatLngBounds fit)
- 장소가 없으면 서울 기본값 유지
- 핀 클릭 → 하단 상세 패널 슬라이드업
- 지도 클릭으로 핀 추가 불가 (click 이벤트 리스너 제거)

---

## 핀 상세 패널

핀 클릭 시 하단에서 슬라이드업:

```
장소명                          ×
📅 방문일
──────────────────────────────
[사진 (있을 경우)]
메모 텍스트
──────────────────────────────
[수정]                    [삭제]
```

---

## 장소 추가 흐름

1. 지도 상단 검색창에 키워드 입력 → 카카오 키워드 검색 API 호출
2. 드롭다운 결과에서 선택 → 지도 해당 위치로 이동 + 미리보기 핀 표시
3. 하단 추가 폼 패널 열림: 장소명(자동), 메모, 방문일, 사진
4. 저장 → DB에 저장, 미리보기 핀 → 정식 핀으로 전환

---

## 변경 범위

- `components/TravelContent.tsx` 리팩토링
  - 사이드바 UI 추가 (desktop)
  - 바텀시트 UI 추가 (mobile)
  - 지도 클릭 이벤트 제거
  - fit bounds 로직 추가
  - 핀 상세 패널에 사진 표시 추가
- API/DB 스키마 변경 없음
- `TripFormSheet.tsx` 변경 없음

---

## 유지되는 것

- Trip / TripPlace 데이터 모델
- `/api/trips` 엔드포인트
- TripFormSheet 컴포넌트
- 카카오맵 초기화 패턴 (`autoload=false` + `maps.load()`)
